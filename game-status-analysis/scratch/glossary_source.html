Production workflow

A pull system across two boards. Each team pulls work in when it has capacity — nobody pushes downstream. Every stage shows its owner; tap a role to highlight where it acts; click any stage for its detail.

01

Pull, not push

When a stage's team has free capacity, it pulls in games from the previous stage whose Definition of Done is met and that satisfy this stage's Definition of Entry. Nobody pushes work forward.

02

PM watches the whole Game board

The PM is a watcher on every Game-board stage, and the assignee on the coordination-heavy ones — To Do, Contract Alignment, Development, Integration QC, Packaging.

03

WIP & time

Every stage has a WIP limit (n) and must track ETA + actual start/end.

04

One epic per game

A game = one epic. Child tickets link to it on the Verticals/Studios boards. Closed when dev is done + all tickets, docs and time are in.

Epic structure — how a game's Jira issues link up

A game is tracked as one epic. Each team builds its part as a child ticket on its own board, and links that ticket back to the game epic.

The epic is the game. Its assignee changes per stage to whoever owns it (the PM on coordination stages); the epic holds the source-of-truth documents.
