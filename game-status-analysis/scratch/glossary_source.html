Epic structure — how a game's Jira issues link up

A game is tracked as one epic. Each team builds its part as an issue on its own board, and links that issue back to the game epic.

The epic is the game. The PM is the assignee and owns it end-to-end; the epic holds the source-of-truth documents.

Each task is created on its own board and linked to the game epic. The epic closes only when development is done and every task, document and timestamp is linked — so the finished game stands as a complete record (storage).
